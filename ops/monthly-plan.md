# 初月行動計画（14〜30日で初受注）

目標：30日以内に初受注。最低月5万円。
前提：匿名運営・固定費ゼロ・営業はX DMとメールが中心。

## 週次ゴール

| 週 | ゴール |
|---|---|
| Week 1 | LP公開・問い合わせフォーム稼働・デモ動画撮影 |
| Week 2 | X運用開始・営業リスト100件作成・DM/メール50件着地 |
| Week 3 | 追加DM/メール50件・問い合わせ対応・見積送付 |
| Week 4 | 初受注→納品・事例化許可を取る・次月リストに繋ぐ |

## Day別アクション

### Day 1
- [ ] 屋号を決める（`sales/positioning.md` のサービス名で兼用可）
- [ ] 連絡用メールアドレス（屋号@gmail.com など）を作成
- [ ] 振込先口座を決める（個人口座で開始、軌道に乗ったら屋号口座）
- [ ] X アカウントを作成 or 既存アカウントを切り替え

### Day 2
- [ ] GitHub Pages に LP 公開（`docs/` を Pages 化）
- [ ] Googleフォームを作成（`docs/inquiry-form-setup.md`）
- [ ] LPの `INQUIRY_URL` に反映・動作確認
- [ ] LP末尾の「屋号 / 連絡先」を実値に置換
- [ ] 特商法表記・プライバシーポリシーのリンクを設置

### Day 3
- [ ] デモ用スプレッドシートを作成（`deliverables/sample-data.md`）
- [ ] Before/After スクリーンショットを撮影
- [ ] 30秒デモ動画を撮影（画面収録→LP直下に埋め込み or YouTube限定公開）
- [ ] X プロフィール・固定投稿を整備（`x-social/`）

### Day 4〜5
- [ ] 営業リスト作成（`ops/outreach-list-guide.md`）
  - 大学サークル X アカウント 30件
  - イベント運営アカウント 30件
  - 小規模事業者 X アカウント 20件
  - PR TIMES / 自治体 / 商工会 経由の候補 20件
- [ ] リストをスプレッドシートで管理

### Day 6〜7
- [ ] Xで役立つ投稿を3本投稿（`x-social/content-themes.md` 参照）
- [ ] 固定投稿を設定
- [ ] 準備日の終わり

### Day 8〜14（Week 2）
- [ ] DM 送信：1日10件 × 5日 = 50件
  - `sales/outreach-x-dm.md` をベースに、相手ごとに1行カスタム
- [ ] 並行で X に発信を続ける（週3〜5投稿）
- [ ] 問い合わせが来たら T2 テンプレで即返信
- [ ] 金曜夜に週次振り返り：返信数 / 見積提示数

### Day 15〜21（Week 3）
- [ ] 追加 DM/メール 50件
- [ ] 1件目の問い合わせが来たら即ヒアリング→見積→入金案内
- [ ] 見送りの理由があれば記録（次週の改善材料）
- [ ] LP を反響に合わせて微修正（特に FAQ）

### Day 22〜30（Week 4）
- [ ] 初受注・作業開始
- [ ] 納品後フォロー T6
- [ ] **お客様の許可を得て事例化**（匿名化）
  - 「どんな課題があり、どう解決したか」を2〜3行で
  - 次月のLPに「こんな現場で使われています」実事例として追加
- [ ] 2件目のリード育成

## 判断ポイント

| 状況 | 判断 |
|---|---|
| DMから返信ゼロ（50件送って） | 文面を見直し（特に1文目の相手言及） |
| 見積出しても成約しない | 価格ではなく「不安」。保証・手順・権限の説明を追加 |
| 問い合わせはあるが単価が合わない | ライトを主軸にして数を捌く or カスタム要件を拾う |
| 作業量が見積より増える | 見積書に「対応範囲外の例」を明記 |

## 初月のNG

- LPに詰め込みすぎる（初版は8割で出す。反響で直す）
- DMの一斉送信（1通1通カスタムしないと返信率が一桁台に落ちる）
- 「無料で試します」（単価の基準が壊れる）
- 納期の約束を甘くする（7日→14日にずれる想定で仕事を受けない）
