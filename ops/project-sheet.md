# 案件管理シート設計

Googleスプレッドシートで案件を管理する際の列定義。10件超えたら記憶では回せない前提で設計。

## シート1: `案件一覧`

| 列 | 項目 | 形式 | 必須 | 備考 |
|---|---|---|---|---|
| A | 案件ID | テキスト | ○ | `YYMMDD-01` 形式。受信日ベース |
| B | 受信日 | 日付 | ○ | 問い合わせが来た日 |
| C | お客様名 | テキスト | ○ |  |
| D | 所属 | 選択 | ○ | サークル/研究室/イベント/小規模事業者/個人事業主/その他 |
| E | メール | テキスト | ○ |  |
| F | プラン | 選択 | ○ | ライト/スタンダード/カスタム/未確定 |
| G | 金額 | 数値 | △ | 見積確定後 |
| H | ステータス | 選択 | ○ | 下記参照 |
| I | 見積送付日 | 日付 | △ |  |
| J | 入金日 | 日付 | △ |  |
| K | 着手日 | 日付 | △ |  |
| L | 納品日 | 日付 | △ |  |
| M | サポート期限 | 日付 | △ | 納品日 +30 or +60 |
| N | 備考 | テキスト | △ | 特殊要件、やり取り要点 |
| O | フォローアップ予定日 | 日付 | △ | 3日後・27日目など |

### ステータス値

- `受信`
- `ヒアリング中`
- `見積送付済`
- `入金待ち`
- `作業中`
- `納品済`
- `サポート中`
- `完了`
- `辞退`
- `返金`

## シート2: `売上`

月次で集計するための粗いログ。

| 列 | 項目 |
|---|---|
| A | 入金日 |
| B | 案件ID |
| C | 金額 |
| D | プラン |
| E | 手数料（振込手数料など） |
| F | 実入金額 |

## シート3: `タスク`

今日やるべきタスク。受注後の進行管理用。

| 列 | 項目 |
|---|---|
| A | 案件ID |
| B | 期限 |
| C | タスク内容 |
| D | ステータス（未/進行中/完了） |

## 運用ルール

- 問い合わせが来たら、まず `案件一覧` に1行追加
- ステータス更新は即時（後でまとめて、は必ず漏れる）
- フォローアップ予定日は「納品日＋3」「納品日＋27」を自動計算で埋める
- 月初に前月の `売上` を集計し、月5万円の達成状況を確認
- `案件ID` はメール件名の末尾に `[YYMMDD-01]` の形で付けると過去スレッド検索がラク

## KPI（初受注までの判断材料）

| 指標 | 目安 |
|---|---|
| DM送信→返信率 | 3%以上（30件で1件返信が平均下限） |
| 返信→見積提示率 | 50%以上 |
| 見積提示→受注率 | 30%以上 |

→ 月5万円（スタンダード2件 or ライト＋スタンダード）の場合、
逆算で DM/メール ≒ 200件 前後／月 が初月の動き出しライン。
