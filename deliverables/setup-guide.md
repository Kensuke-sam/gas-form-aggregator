# 運用マニュアル（納品物）

このドキュメントは「フォーム仕分け自動化パック」の納品時にお渡しする、お客様向け運用マニュアルです。PDFに書き出して納品してください。

---

## 1. 仕組みの全体像

```
[Googleフォーム] → [回答シート] → (自動仕分け) → [カテゴリ別シート]
                                      ↓
                              [通知メール / Slack / Chatwork]
                                      ↓
                                   [_log]
```

- フォームに回答が届くと、あらかじめ決めたルールで判定
- 該当するシートへ自動で並ぶ
- 必要なら通知が飛び、処理履歴が `_log` に残る

## 2. 設定シートの使い方

スプレッドシートには納品時に以下3つのシートが作成されています。

| シート | 役割 | お客様が触るか |
|---|---|---|
| `_config` | 基本設定 | ○（必要時のみ） |
| `_rules` | 仕分けルール | ○（運用で触る） |
| `_normalize` | 表記揺れ吸収マップ（任意） | ○（必要に応じて） |
| `_log` | 処理履歴 | 閲覧のみ |
| `_dashboard` | 集計サマリー（生成時のみ作成） | 閲覧のみ |

### `_config` の主要項目

| key | 説明 |
|---|---|
| `source_sheet` | フォーム回答が入るシート名 |
| `default_target` | どのルールにも合わなかった回答の行き先（空ならスキップ） |
| `dedupe_columns` | 重複検知に使う列番号（カンマ区切り 例: 2,4 / 空なら無効） |
| `notify_enabled` | メール通知の有効/無効 |
| `slack_webhook` | Slack通知先URL（空なら送信なし） |
| `log_enabled` | 処理履歴の記録有無 |

### `_rules` の書き方

| 列 | 内容 | 例 |
|---|---|---|
| `rule_name` | 人が読むためのラベル | 資料請求 |
| `match_column` | 判定に使う列番号（A列=1） | 3 |
| `operator` | 判定方法 | equals |
| `value` | 比較値 | 資料請求 |
| `target_sheet` | 振り分け先のシート名 | 資料請求一覧 |
| `notify_email` | 通知メールの送信先（不要なら空） | sales@example.com |
| `slack_mention` | Slack通知でのメンション（不要なら空） | @channel |
| `有効` | TRUE / FALSE | TRUE |

**ルールは上から順に判定され、最初にマッチしたものが採用されます。**

### 使える operator 一覧

| operator | 意味 | 例 |
|---|---|---|
| `equals` | 完全一致 | 「資料請求」と一致 |
| `not_equals` | 不一致 |  |
| `contains` | 部分一致 | 「@gmail.com」を含む |
| `not_contains` | 部分不一致 |  |
| `starts_with` | 先頭一致 | 「急」で始まる |
| `ends_with` | 末尾一致 | 「株式会社」で終わる |
| `regex` | 正規表現 | `^[0-9]{10}$` |
| `gte` / `lte` | 数値の 以上／以下 | 500000 以上 |
| `gt` / `lt` | 数値の より大／より小 |  |
| `eq` | 数値の等価 |  |
| `any` | 常にマッチ（フォールバック用） |  |

### `_normalize` の使い方（任意）

フォームで自由記述を受けている場合、表記揺れ（「資料要求」「資料請求」「しりょうせいきゅう」）がルール判定の邪魔になります。`_normalize` に「この値が来たら、この値に置き換える」を書いておくと、仕分け判定の前に自動で統一されます。

| column | raw | canonical | 有効 |
|---|---|---|---|
| 3 | 資料要求 | 資料請求 | TRUE |
| 3 | 見積り | 見積もり | TRUE |

## 3. よく使う操作

### 全件を一括で仕分け直したい

メニュー `フォーム仕分け → 全件を一括仕分け`

既存のカテゴリシートはクリアされ、最新のルールで作り直されます。

### 実行前に結果だけ確認したい（ドライラン）

メニュー `フォーム仕分け → ドライラン（書き込みなし）`

既存データに対して現在のルールを適用した場合の「どのルールが何件マッチするか」だけを表示します。書き込みはしません。新しいルールを追加した後の検算に便利です。

### 集計を見たい

メニュー `フォーム仕分け → ダッシュボードを再生成`

`_dashboard` シートに、カテゴリ別・ルール別・日付別の件数が出力されます。

### 新しいルールを追加したい

1. `_rules` シートに1行追加
2. メニュー `フォーム仕分け → ドライラン` で結果確認
3. メニュー `フォーム仕分け → テスト実行（最新1件）` で動作確認
4. 次のフォーム送信から自動で適用

### 重複申込を検知したい

`_config` の `dedupe_columns` に、重複判定のキーにする列番号をカンマ区切りで入力します。
例：`2,4` → B列（メールアドレス）とD列（希望日程）が同じ回答は重複扱い。

### 通知が来なくなった

1. `_config` の `notify_enabled` が `true` になっているか
2. `_rules` の該当行に `notify_email` が設定されているか
3. `_log` に `error` 行がないか

## 4. トラブル時の確認箇所

| 症状 | まず見る場所 |
|---|---|
| 仕分けされない | `_log` の `status` 列 |
| 通知が来ない | `_config` の `notify_enabled` と `_rules` の `notify_email` |
| 想定外のシートに入る | `_rules` のルール順序（上から順に判定される） |
| シートが増えすぎた | `_rules` の `target_sheet` 表記ゆれ（半角/全角・スペース） |
