# 仕分けルール リファレンス

納品後にお客様自身でルールを追加・編集できるよう、`_rules` シートの仕様をまとめます。

## 列の意味

| 列 | 必須 | 内容 |
|---|---|---|
| `rule_name` | ○ | ルールの表示名（ログにも出力される） |
| `match_column` | ○ | 判定対象の列番号（A列=1, B列=2...） |
| `operator` | ○ | 判定方法。下記表を参照 |
| `value` | △ | 比較値。`any` / `default` の場合は不要 |
| `target_sheet` | ○ | 振り分け先のシート名 |
| `notify_email` | × | 通知メール送信先（カンマ区切りで複数可） |
| `slack_mention` | × | Slack通知に付与するメンション |
| `有効` | × | TRUE / FALSE。空欄はTRUE扱い |

## operator 対応表

| operator | データ型 | 意味 | `value` の例 |
|---|---|---|---|
| `equals` | 文字列 | 完全一致（前後空白は無視） | `資料請求` |
| `not_equals` | 文字列 | 不一致 | `テスト` |
| `contains` | 文字列 | 部分一致 | `株式会社` |
| `not_contains` | 文字列 | 部分不一致 | `@example.com` |
| `starts_with` | 文字列 | 先頭一致 | `急` |
| `ends_with` | 文字列 | 末尾一致 | `御中` |
| `regex` | 文字列 | 正規表現マッチ | `^[0-9]{3}-[0-9]{4}$` |
| `gte` | 数値 | 以上 | `500000` |
| `lte` | 数値 | 以下 | `10` |
| `gt` | 数値 | より大 | `0` |
| `lt` | 数値 | より小 | `100` |
| `eq` | 数値 | 等価 | `1` |
| `any` | ― | 常にマッチ（フォールバック） | 空でOK |

## ルール順序の考え方

- ルールは上から順に評価
- 最初にマッチしたものだけが適用される
- 「特殊ケース → 一般ケース → フォールバック」の順で並べる

### 例：金額別に分ける

```
1. rule_name: 高額案件  | match_column: 4 | operator: gte | value: 500000 | target_sheet: 優先対応
2. rule_name: 中額案件  | match_column: 4 | operator: gte | value: 100000 | target_sheet: 通常対応
3. rule_name: それ以外  | match_column: 4 | operator: any |              | target_sheet: 低額対応
```

この順序を入れ替えると、すべての回答が「低額対応」に吸い込まれるので注意。

## よくあるハマりどころ

- **全角/半角**：`equals` では半角と全角は別物扱い
- **前後空白**：`equals` は `trim` するが、`contains` は `trim` しない
- **数値判定**：セルがテキスト形式だと `gte` が動かないことあり。セル書式を数値に
- **シート名の表記ゆれ**：`資料請求 `（末尾スペース）と `資料請求` は別シート扱い
- **`default_target`**：どのルールにもマッチしないとき、`_config` の `default_target` に送られる。未設定だとスキップ（`_log` に `skipped` が残る）
