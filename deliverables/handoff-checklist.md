# 納品時チェックリスト（社内用）

納品前に必ず全項目を確認する。チェック漏れによる「動かない納品」を防ぐためのもの。

## 作業開始前

- [ ] お客様のGoogleアカウントへ編集権限を受け取った
- [ ] 連絡用メールアドレスを控えた
- [ ] プランを確認した（ライト / スタンダード / カスタム）
- [ ] 仕分けたいカテゴリの一覧を受領した
- [ ] フォームのURLを受領した
- [ ] ご入金を確認した

## 作業中

- [ ] `Code.gs` をスクリプトエディタに貼り付けた
- [ ] `initSetupSheets` を実行し、`_config` / `_rules` / `_log` シートが生成された
- [ ] `_config` の `source_sheet` が実際の回答シート名と一致している
- [ ] `_rules` に仕分けルールを入力した
- [ ] スタンダード以上：`notify_email` を設定した
- [ ] カスタム：`slack_webhook` または Chatwork 設定を入力した

## 動作確認

- [ ] テスト送信1：該当ルールにマッチする回答を送信 → 該当シートに行が追加される
- [ ] テスト送信2：マッチしないケース → `default_target` に入る／ログに `skipped`
- [ ] 通知メールが届く（該当プラン）
- [ ] Slack/Chatwork 通知が飛ぶ（カスタム）
- [ ] `_log` に `success` ログが記録されている
- [ ] `sortAllResponses` を実行して既存データが正しく仕分けされる

## 納品物

- [ ] 運用マニュアルPDF（`setup-guide.md` から出力）
- [ ] ルールリファレンスPDF（`rule-reference.md` から出力）
- [ ] テスト送信のスクリーンショット（Before/After）
- [ ] 設定シートの記入済みコピー
- [ ] 「権限返却の手順書」

## 納品後

- [ ] 編集権限を返却（お客様のアカウントから自分を削除してもらう）
- [ ] サポート期間終了日を記録（30日 / 60日）
- [ ] 請求書／領収書を送付
- [ ] 納品から3日後、4週目にフォローメール
