# 動作確認チェックリスト（GASエディタ上での手動テスト）

Code.gs を本番スプレッドシートへ貼り替えた際に実施する検証項目。
コード変更時は毎回、該当する項目のみでも通しておく。

---

## 0. 事前準備

- [ ] 対象スプレッドシートを1つ用意（空の新規ブックでOK）
- [ ] `拡張機能 → Apps Script` を開く
- [ ] 既存コードを全削除し、`Code.gs` の中身を貼り付けて保存
- [ ] スプレッドシートを再読み込み → メニュー「フォーム仕分け」が表示されることを確認

## 1. 初期化まわり

- [ ] `フォーム仕分け → 設定シートを初期化` を実行
  - `_config` / `_rules` / `_normalize` / `_log` の4シートが作成される
  - 2回目の実行で**エラーにならず**、シートが重複作成されない（`ensureXxxSheet` の冪等性）

## 2. デモデータ投入 → 一括仕分け

- [ ] `フォーム仕分け → デモデータを投入` を実行
  - 「フォームの回答 1」シートに10件投入され、`_rules` がデモ用に差し替わる
- [ ] `フォーム仕分け → 全件を一括仕分け` を実行
  - `学生申込` / `関係者申込` / `団体対応` / `4月20日分` シートが作られる
  - 各シートの**1行目にヘッダ行が復元**されている（青背景・白文字・固定行）
  - 2行目以降にデータが入っている
- [ ] もう一度「全件を一括仕分け」を実行
  - `clearContents()` によって既存内容がクリアされ、最新ルールで**再生成**される
  - **他のシート（`_config` / `_rules` / `_normalize` / `_log` / フォームの回答 1）は無傷**
  - 追加で手動作成したシート（仕分け対象外）も無傷

## 3. 既存データを壊さない保証

- [ ] 仕分け先シート（例: `学生申込`）に、**手動で E 列より右に「メモ」列**を追加してデータを入れる
- [ ] 再度「全件を一括仕分け」を実行
  - `clearContents()` はシート全体の**値のみを**クリアするため、メモ列の値も消える
  - ※**これは仕様**。目的シートに手書きメモを残したい場合は別シートへ退避するよう顧客へ案内する
- [ ] システムシート（`_config` 等）の値は保持されているか再確認

## 4. header_row = 2 の設定

想定ケース: フォームの回答シートの1行目に備考行があり、2行目がヘッダの場合。

- [ ] `_config` の `header_row` を `2` に変更
- [ ] 「フォームの回答 1」の1行目（現在のヘッダ）の上に新しい行を挿入し、1行目は空欄のまま、2行目に元のヘッダを移動
- [ ] `フォーム仕分け → 全件を一括仕分け` を実行
  - `data.slice(config.headerRow)` が**行3以降**を対象にし、ヘッダが2行目から取られる
  - 各カテゴリシートに**2行目から始まる正しいヘッダ**が書き戻される
- [ ] `フォーム仕分け → テスト実行（最新1件）` を実行
  - `source.getRange(lastRow, 1, 1, source.getLastColumn())` で最新行が取得され、
    正しいルールが適用される

## 5. onFormSubmit のテスト

実フォーム接続前でも、エディタ上から模擬 `e.values` で検証可能。

- [ ] Apps Script エディタで以下を一時的に追加し実行：

```javascript
function __testOnSubmit() {
  onFormSubmit({
    values: [new Date(), 'test@example.com', '学生', '4/20', 1]
  });
}
```

- [ ] `学生申込` シートに1行追加されること
- [ ] `_log` に `status = success` / `matched_rule = 学生` の行が記録されること
- [ ] 同じ `e.values` で再実行（`dedupe_columns` を `2` に設定した上で）
  - 2回目は `_log` に `status = duplicate` が記録され、シートへは追記されない

## 6. ドライラン

- [ ] `フォーム仕分け → ドライラン（書き込みなし）` を実行
  - カテゴリシートには**何も書き込まれない**
  - ダイアログに件数サマリが表示される
  - `_log` に `status = dry-run` が記録される

## 7. ダッシュボード

- [ ] `フォーム仕分け → ダッシュボードを再生成` を実行
  - `_dashboard` シートに「カテゴリ別 / ルール別 / 日付別」の3セクションが出る
  - 2回目の実行で内容が**正しく再生成**される（前回の古い値が残らない）

## 8. エラーハンドリング

- [ ] `_config` の `source_sheet` を存在しないシート名に変更 → 「全件を一括仕分け」
  - ダイアログに「ソースシートが見つかりません：...」と表示される
  - `_log` に `status = error` が記録される
- [ ] 値を元に戻す

## 9. トリガー

- [ ] `フォーム仕分け → 自動トリガーを設定` → 完了ダイアログ
- [ ] `Apps Script → トリガー` メニューから `onFormSubmit` が1件登録されていることを確認
- [ ] 同操作を2回実行しても**トリガーが重複しない**（既存 `onFormSubmit` を削除してから再作成）
- [ ] `フォーム仕分け → 自動トリガーを解除` → ダイアログ「解除しました（1件）」

## 10. 最終片付け

- [ ] テスト用シート・テスト用トリガーを削除
- [ ] `_log` を必要に応じてクリア

---

## 自動テスト（補助）

ルール判定ロジックは Node.js 側の `tests/rule-engine.test.js` でユニットテスト済み。
`Code.gs` を編集したら `lib/rule-engine.js` も同期更新し、以下を実行：

```bash
npm test
```

すべてパスすることを確認してから GAS エディタでの手動確認を行う。
