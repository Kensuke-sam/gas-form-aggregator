# データと権限の取り扱い（お客様向け）

匿名運営に対する不安を下げるため、データと権限の扱いを明文化します。

## データの流れ

```
[お客様のフォーム] → [お客様のスプレッドシート]
              ↓
        [お客様のGASプロジェクト（同一アカウント内）]
              ↓
         (設定に応じて)
         ├→ [お客様が指定したメールアドレス]
         ├→ [お客様が指定したSlack/Chatwork]
         └→ [お客様のスプレッドシート内 _log]
```

- スクリプトはお客様のGoogleアカウント内で動作します
- 外部サーバーへのデータ送信は行いません（Slack/Chatwork連携を有効にした場合のみ、指定の連携先にデータが送られます）
- 当方がお客様のデータを受け取る経路はありません

## 権限の扱い

- 作業時：お客様のスプレッドシートへの「編集権限」を一時的にいただきます
- 作業範囲：`Code.gs` の設置、設定シートの作成、トリガーの設定、テスト送信の確認
- 作業対象外：他のフォルダ・ファイル・Gmail・カレンダー等は一切触れません
- 納品後：権限返却の手順書をお渡しし、編集権限を返却します

## 権限返却の手順（納品時にお客様に共有）

1. スプレッドシートを開く
2. 右上「共有」ボタンをクリック
3. 共有先一覧から当方のアカウントを選択
4. 「削除」を選ぶ

## Slack / Chatwork 連携について（カスタムプラン）

- 通知はお客様がご用意いただいた Webhook URL / API トークンを使います
- 認証情報は `_config` シート内に保存されます（外部共有されません）
- シートの共有範囲には十分ご注意ください

## GAS の権限スコープ

本スクリプトが利用するGoogleサービスの権限：

- スプレッドシート（読み書き）
- メール送信（通知メール）
- 外部URLアクセス（Slack / Chatwork 連携を使う場合のみ）
- トリガー管理（フォーム送信時の自動実行）

## 返金保証

- 納品後7日以内に動作確認ができない場合、全額返金します
- 返金条件：お客様側の設定変更やフォーム構造変更が原因でないこと
