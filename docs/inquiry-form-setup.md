# 問い合わせフォームのセットアップ手順

LPのCTAから遷移する先のGoogleフォームを、そのまま使える形で定義します。

## 1. Googleフォームを作成

1. <https://forms.google.com/> から新規フォームを作成
2. タイトル：`フォーム仕分け自動化パック / 現状ヒアリング`
3. 説明：下記を貼り付け

```
Googleフォームの仕分け自動化について、現在の状況を教えてください。
回答をもとに、24時間以内にプラン提案と見積もりをお送りします。
見積もりまでは無料です。ご検討だけでも歓迎します。
```

## 3. 回答先の設定

1. Googleフォームの「回答」タブを開く
2. スプレッドシートアイコンをクリックして回答シートを作成
3. 作成されたスプレッドシートをブックマーク

## 4. 自分への通知設定

Googleフォームの「回答」タブ右上の三点メニュー → 「新しい回答についてのメール通知を受け取る」をオンにする。

## 5. 自動返信メッセージの設定

### 5-1. 送信者控えメールの配信（任意）

`設定` タブ → `回答`（Responses）セクション：
- `メールアドレスを収集する` を「確認済み」または「回答者からの入力」に設定
- `回答のコピーを回答者に送信` を「常に」に設定

この2つで、送信した本人に自動で控えメールが届く（件名・本文は Google 側の定型文で固定）。

### 5-2. 送信直後の確認画面メッセージ（推奨）

件名・本文はカスタマイズ不可だが、送信完了画面に表示されるメッセージは編集できる。
`設定` タブ → `プレゼンテーション`（Presentation）セクション → `確認メッセージ` に、`ops/form-confirmation-message.md` の本文を貼り付ける。

## 6. LPに反映

`docs/index.html` 末尾のスクリプト内、`INQUIRY_URL` にフォームの公開URLを設定する：

```js
const INQUIRY_URL = 'https://forms.gle/xxxxxxxxx';
```

未設定の間は、LP 最下部の CTA ボタンが `CONTACT_EMAIL` への mailto リンクにフォールバックする。
`contact@example.com` は仮置きなので、本番公開前に実メールに差し替えること（`config/site-values.json` を編集 → `npm run site:apply` で一括反映できる）。

## 7. 動作確認

- LPのCTAボタンをクリック → 新しいタブでフォームが開く
- 自分で1件テスト送信
- メール通知が届く
- 回答シートに記録されていることを確認

## 8. 運用

- 回答が来たら、24時間以内にプラン提案＋見積もりを返信
- 返信テンプレは `ops/email-templates.md` の T2（ヒアリング）→ T3（見積もり）を順に使う
